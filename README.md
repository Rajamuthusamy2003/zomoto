Zomato Restaurant Data Analysis using MySQL
Project Overview

Performed Exploratory Data Analysis (EDA) on a Zomato dataset containing 9,000+ restaurant records across 15 countries using MySQL. The objective was to clean, transform, and analyze restaurant data to uncover insights related to customer preferences, restaurant distribution, ratings, cuisines, and service offerings.

Key Tasks Performed
Analyzed table structure, data types, and constraints.
Identified and validated duplicate records using RestaurantID.
Cleaned data by removing redundant columns and correcting inconsistent city names.
Merged datasets using CountryCode to add country information.
Applied aggregate and window functions to analyze restaurant trends.
Calculated key metrics such as ratings, votes, and pricing statistics.
Created custom rating categories for enhanced analysis.
Key Insights
India accounted for 90.67% of restaurants in the dataset, followed by the USA (4.45%).
Only India and UAE provided online delivery services, with adoption rates of 28.01% and 46.67%, respectively.
Connaught Place, New Delhi had the highest number of restaurants (122), followed by Rajouri Garden (99) and Shahdara (87).
North Indian Cuisine was the most popular cuisine in Connaught Place.
Restaurants offering table booking had a higher average rating (3.9/5) compared to those without table booking (3.7/5).
Identified India Restaurant (Restaurant ID: 20747, Kolkata) as the best-performing moderately priced restaurant based on ratings, votes, online delivery, and table booking availability.
Technologies Used

MySQL, SQL, Joins, Window Functions, Data Cleaning, Data Transformation, Exploratory Data Analysis (EDA)

Outcome

Generated actionable business insights on restaurant performance, customer preferences, cuisine trends, and service adoption using MySQL-based data analysis techniques.
